3.1 GRID TRADING ALGORITHM
--------------------------------------------------------------------------------

PURPOSE:
The Grid Trading algorithm is designed to profit from normal price oscillations
in ranging/sideways markets. It places multiple buy and sell orders at fixed
price intervals, capturing profit as price bounces between levels.

HOW IT WORKS:

Step 1: Grid Initialization
   - Calculate center price from current market price
   - Define grid levels above and below center
   - Calculate spacing between each level (default: 0.5%)

Step 2: Order Placement
   - Place BUY orders BELOW current price (catch dips)
   - Place SELL orders ABOVE current price (catch pumps)
   - Each order has equal USD value (default: $10)

Step 3: Order Execution Flow
   When a BUY order fills:
      â†’ Price dropped to our buy level
      â†’ We now hold a position
      â†’ The corresponding SELL order above will close it for profit
   
   When a SELL order fills:
      â†’ Price rose to our sell level
      â†’ We captured the grid profit
      â†’ System places new orders to maintain the grid

MATHEMATICAL MODEL:

   Grid Levels = Center Price Ã— (1 Â± Spacing Ã— Level_Number)

   Example for BTC at $95,000, 3 levels, 0.5% spacing:
   
   SELL LEVELS:              BUY LEVELS:
   Level 3: $96,425 (+1.5%)  Level 3: $93,575 (-1.5%)
   Level 2: $95,950 (+1.0%)  Level 2: $94,050 (-1.0%)
   Level 1: $95,475 (+0.5%)  Level 1: $94,525 (-0.5%)
                    
                    CENTER: $95,000

   Potential profit per cycle: 0.5% - 1.5% depending on volatility

FILTER INTEGRATION:
The Grid algorithm uses RSI/MACD filters to avoid placing grids in:
- Strongly trending markets (where price might break through all levels)
- Extremely overbought/oversold conditions
- High volatility periods (detected by unusual candle sizes)

CODE IMPLEMENTATION:
   
   def calculate_grid_levels(self, center_price: float):
       levels = self.config['grid_levels']
       spacing = self.config['grid_spacing_percent'] / 100
       
       buy_levels = []
       sell_levels = []
       
       for i in range(1, levels + 1):
           buy_price = center_price * (1 - spacing * i)
           sell_price = center_price * (1 + spacing * i)
           buy_levels.append(buy_price)
           sell_levels.append(sell_price)
       
       return {'buy': buy_levels, 'sell': sell_levels}

4.1 RSI (RELATIVE STRENGTH INDEX)
--------------------------------------------------------------------------------

WHAT IT MEASURES:
RSI measures the speed and magnitude of recent price changes to evaluate
overbought or oversold conditions. Range: 0 to 100.

CALCULATION:
                        100
   RSI = 100 - â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
                1 + (AvgGain/AvgLoss)

   Where:
   - AvgGain = Average of all UP moves in last N periods
   - AvgLoss = Average of all DOWN moves in last N periods
   - N = 14 (standard period)

SIGNAL INTERPRETATION:
   RSI > 70  â†’ OVERBOUGHT â†’ Potential SELL signal
   RSI < 30  â†’ OVERSOLD   â†’ Potential BUY signal
   RSI 40-60 â†’ NEUTRAL    â†’ No clear signal

OUR ENHANCED IMPLEMENTATION:
We calculate signal strength based on how extreme the RSI is:

   def calculate_rsi(self, prices, period=14):
       deltas = [prices[i] - prices[i-1] for i in range(1, len(prices))]
       gains = [d if d > 0 else 0 for d in deltas]
       losses = [-d if d < 0 else 0 for d in deltas]
       
       avg_gain = sum(gains[-period:]) / period
       avg_loss = sum(losses[-period:]) / period
       
       if avg_loss == 0:
           return 100.0
       
       rs = avg_gain / avg_loss
       rsi = 100 - (100 / (1 + rs))
       
       # Signal strength calculation
       if rsi >= 70:
           strength = min((rsi - 70) * 3.33, 100)  # Stronger at extremes
       elif rsi <= 30:
           strength = min((30 - rsi) * 3.33, 100)
       else:
           strength = 50 - abs(50 - rsi)
       
       return rsi, strength

4.2 MACD (MOVING AVERAGE CONVERGENCE DIVERGENCE)
--------------------------------------------------------------------------------

WHAT IT MEASURES:
MACD shows the relationship between two exponential moving averages (EMAs).
It reveals changes in trend strength, direction, momentum, and duration.

COMPONENTS:
- MACD Line:    Fast EMA (12) - Slow EMA (26)
- Signal Line:  9-period EMA of MACD Line
- Histogram:    MACD Line - Signal Line

SIGNAL INTERPRETATION:
   MACD crosses ABOVE Signal â†’ BULLISH crossover â†’ BUY signal
   MACD crosses BELOW Signal â†’ BEARISH crossover â†’ SELL signal
   Histogram growing         â†’ Momentum increasing
   Histogram shrinking       â†’ Momentum decreasing

COMBINED WITH RSI:
We use MACD to CONFIRM RSI signals:
- RSI Oversold + MACD Bullish Crossover = STRONG BUY
- RSI Overbought + MACD Bearish Crossover = STRONG SELL
- Conflicting signals = WAIT

6.3 DYNAMIC POSITION SIZING
--------------------------------------------------------------------------------

Position size adapts to signal quality and account status:

BASE CALCULATION:
   Position Size = (Account Balance Ã— Risk %) / Stop Loss Distance

DYNAMIC ADJUSTMENTS:

   Win Streak (3+ consecutive wins):
      - Increase size by 25% (confidence is high)
      - Cap at 150% of normal size
   
   Loss Streak (3+ consecutive losses):
      - Decrease size by 50% (protect capital)
      - Minimum 25% of normal size
   
   High Volatility:
      - Decrease size by 25% (wider stops needed)
   
   Low Volatility:
      - Can increase size slightly (tighter stops possible)

KELLY CRITERION (Advanced):
For optimal growth, we consider the Kelly formula:

   f* = (p Ã— b - q) / b
   
   Where:
   - f* = fraction of capital to risk
   - p = probability of winning
   - b = ratio of win to loss
   - q = probability of losing (1-p)
   
   Example: 60% win rate, 1.5:1 R:R
   f* = (0.6 Ã— 1.5 - 0.4) / 1.5 = 0.33
   
   We use HALF-KELLY (16.5%) for safety
